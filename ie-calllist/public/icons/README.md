# PWA Icons

## Current Status

This directory contains **SVG placeholder icons** generated by `scripts/generate-icons.ts`.

## Production Requirements

For production deployment, these SVG files must be converted to **PNG format** using the actual Inside Edition logo.

### Required PNG Sizes

- `icon-72x72.png`
- `icon-96x96.png`
- `icon-128x128.png`
- `icon-144x144.png`
- `icon-152x152.png`
- `icon-192x192.png` (maskable)
- `icon-384x384.png`
- `icon-512x512.png` (maskable)

### Conversion Options

1. **Online Converter:**
   - Visit https://convertio.co/svg-png/
   - Upload each SVG file
   - Download as PNG

2. **ImageMagick (Command Line):**
   ```bash
   convert icon-192x192.svg icon-192x192.png
   ```

3. **Design Tool:**
   - Open SVG in Figma/Photoshop/Illustrator
   - Export as PNG at required sizes
   - Use actual IE logo instead of placeholder

### Maskable Icons

The 192x192 and 512x512 icons should be "maskable" - meaning they work well when Android applies various mask shapes. Ensure the logo has proper padding (safe area) around edges.

### Testing

After converting to PNG:
1. Test PWA installation on iOS (Safari)
2. Test PWA installation on Android (Chrome)
3. Verify icons appear correctly on home screen
4. Check that icons scale properly
